# The Element component

Under the hood, every Bulma component renders an `<Element />` component that serves as a layer of abstraction
over plain HTML element.

This layer is required in order to support Bulma helpers/modifiers for every component.
It also gives components flexibility to be rendered as different components. The layer provides every component
a set of common props that you can use to utilize Bulma helpers and customize how components are rendered.

## Bulma helpers

`react-bulma-components` provide support for Bulma helpers in the form of common props among all components.

Unless otherwise specified, **all props described below are available to all components**.

### Color

Bulma color helpers offer the following color variants:

<br />

<Container>
  <Columns>
    {[
      'primary',
      'link',
      'info',
      'success',
      'warning',
      'danger',
      'light',
      'white',
      'dark',
      'white-bis',
      'white-ter',
      'grey-lighter',
      'grey-light',
      'grey',
      'grey-dark',
      'grey-darker',
      'black-ter',
      'black-bis',
      'black',
    ].map((color) => (
      <Columns.Column size={3}>
        <Columns>
          <Columns.Column>{color}</Columns.Column>
        </Columns>

      </Columns.Column>
    ))}
  </Columns>

<br />

To use the light/dark variants of the colors, simply append `-light` or `-dark` at the end of their color name.
For example, `primary-light` corresponds to the light version of the `primary` color.

You can customize colors of components using the following helper props.

| Prop name         | Values               | Description                                 |
| ----------------- | -------------------- | ------------------------------------------- |
| `textColor`       | All supported colors | Changes the text color of a component       |
| `backgroundColor` | All supported colors | Changes the background color of a component |

### Spacing

Spacing helpers are supported via the spacing props. The prop names mimics the naming scheme of the spacing helpers:

- `m*` for **margin** spacing modifiers
- `p*` for **padding** spacing modifiers
- `*t` for **top** spacing (e.g. `mt` modifies top margin)
- `*r` for **right** spacing
- `*b` for **bottom** spacing
- `*l` for **left** spacing
- `*x` for **left and right** spacing
- `*y` for **top and bottom** spacing

Spacing props accepts values from 0 to 6.

| Value | Spacing |
| ----- | ------- |
| 0     | 0       |
| 1     | 0.25rem |
| 2     | 0.5rem  |
| 3     | 0.75rem |
| 4     | 1rem    |
| 5     | 1.5rem  |
| 6     | 3rem    |

### Typography

This table describes all typography helpers

| Prop name       | Values                                                    | Description                                                          |
| --------------- | --------------------------------------------------------- | -------------------------------------------------------------------- |
| `textSize`      | from 1 to 7                                               | Changes text size of component. Can vary depending on viewport.      |
| `textAlignment` | 'centered', 'justified', 'left', 'right'                  | Changes text alignment of component. Can vary depending on viewport. |
| `textTransform` | 'capitalized', 'lowercase', 'uppercase'                   | Transforms case of text of component                                 |
| `textWeight`    | 'light', 'normal', 'semibold', 'bold'                     | Changes font weight of text of component                             |
| `textFamily`    | 'sans-serif', 'monospace', 'primary', 'secondary', 'code' | Changes font family of text of component                             |
| `italic`        | boolean                                                   | Whether to italicize text of component                               |

### Visibility/Responsive helpers

First, the simple props:

| Prop name   | Values                                                   | Description                                                    |
| ----------- | -------------------------------------------------------- | -------------------------------------------------------------- |
| `display`   | 'block', 'flex', 'inline', 'inline-block', 'inline-flex' | Corresponds to display classes in Bulma                        |
| `invisible` | boolean                                                  | Corresponds to CSS: `visibility: hidden`                       |
| `hidden`    | boolean                                                  | Hides the component                                            |
| `srOnly`    | boolean                                                  | Hides component but screen readers can still see the component |

Now, on to the responsive helpers.

Bulma responsive modifiers are available via props for each viewport. Each viewport prop has a special "Only" version
that corresponds to `-only` modifier - the specified modifiers will only take effect at that specific viewport.

For example, mobile viewport modifiers have two props: `mobile` and `mobileOnly`.

Each viewport prop accepts an object of available modifiers that can vary depending on the current viewport.
Each modifier in the object acts the same as their non viewport specific counterpart except their effect depends on the viewport.
The object has the following fields:

| Field name      | Values                                     | Description                                                                |
| --------------- | ------------------------------------------ | -------------------------------------------------------------------------- |
| `display`       | Refer to the `display` modifier prop       | Acts the same as `display` modifier except the effect is viewport specific |
| `hide`          | boolean                                    | Hides the component depending on the viewport                              |
| `textSize`      | Refer to the `textSize` modifier prop      | Changes text size depending on the viewport                                |
| `textAlignment` | Refer to the `textAlignment` modifier prop | Changes text alignment depending on the viewport                           |

As an example, consider the following:

```js
<Container
  mobile={{ hide: true }}
  desktopOnly={{
    display: 'flex',
    textAlignment: 'centered',
  }}
/>
```

The modifier props translate to the following Bulma classes:

- `'is-hidden-mobile'`
- `'is-flex-desktop-only'`
- `'has-text-centered-desktop-only'`

The `Container` will be hidden on `mobile` only;
it will display children in flex and center text on `desktop` viewport **only**.
